아직 시험 적용 중인 것들이 많아서, 내용이 실시간으로 변경될 수 있을 것 같다. 체계와 계획을 정확히 잡고 시작한 것은 아니라서, 더 나은 방향이라거나 다른 방법이 발견되면 바로 수정할 예정이다. md 단위로 정적 컨텐츠를 관리하다보니, 나중에 아카이빙이 쉬워질 것이라는 생각이 든다.

Pelican 선택 배경

SSG(Static Site Generator)로 Pelican을 선택하였다. Pelican이 최신 트렌드를 반영하는 SSG는 아니지만, 테마를 직접 적용하고 커스터마이징하는 과정에서 Python, jinja2 를 활용할 수 있다는 점이 내가 Pelican을 채택한 근거이다.

이렇게는 썼지만, 실제로 직접 커스터마이징할지, 보이는 버그를 commit해서 기여할 수 있을지는 불투명하다. 나중에 빌드 속도 느려지고, 수작업할게 많아진다면 그땐 React나 Vue 기반으로 넘어갈 듯 하다.

주요 예정 컨텐츠

개발 관련 컨텐츠가 주를 이룰 것 같지만, 블로그에 익숙해지다 보면, 다른 이야기도 풀어나갈 수 있을 것 같다. 아무래도 이미지와 링크 활용이 자유로운 블로그이다 보니, 아무래도 정보 제공이 목적인 컨텐츠가 적합하다고 생각한다.

나는 Swift를 PL로 쓰는 iOS개발자이지만, language domain 확장을 위해 1주일에 1~2시간 정도는 서브 PL로써 Python을 1차적으로 활용해볼 계획이다. 당연히 iOS/macOS 개발의 메인 PL인 Swift의 학습/활용 및 개발 테크닉, 설계 방법에 대한 공부가 우선시되어야 할 것이고, 블로그의 메인 개발/성장 컨텐츠도 주로 그 내용이 될 것이다. 그러고나서 남는 시간을 쪼개어 서브 PL로 활용할 Python과 관련된 학습, 컨텐츠 업로드가 이뤄지지 않을까 하는 작은 기대를 해본다.

올해의 목표

단지 주어진 것을 잘 쓰기 보다는, 어떤 환경에서도 나만의 설계/개발 철학으로 자유롭게 개발하는 것이 나의 목표이다.

스타트업과 지금 다니는 회사에서의 경력을 합치면 iOS 개발 경력은 만 4년이 조금 넘는다.

Swift 1.0 -> Objective-C (2016.09 ~ 2018.12) -> 다시 Swift (2019.01~ SSG프로젝트 리뉴얼 시점)

지난 3년 간의 나를 평가해보면, 나는 단지 Cocoa Framework 의 API를 적절히 잘 사용했을 뿐이다. 그마저도 완벽하다고 생각하지 않기 때문에 항상 공부가 필요할 것이다. Swift가 제공하는 Modern language로써의 기능을 적절히 활용할 수 있는가에 대한 나의 대답은 아직 NO이다. 여태 공부했던 지식, 기본 설계 방향을 올해 재정립하여 시니어 개발자의 길을 이어 나가고 싶다. 여기서 더 완성된 형태의 개발자로 성장하기 위해선 API만 잘 활용하는 개발자이기보다는, 구조 설계와 분석에 초점을 두고 성장하는 개발자가 되고 싶다.
